修改容器时间

Date: 2025-11-10  15:57

背景：

测试想单独更改某个服务的时间进行测试流程，需要修改容器时间的同时不影响到宿主机的时间

修改容器时间的方法

  1. 使用 date 命令临时修改 使用 –privileged 和 –cap-add SYS_TIME
    如果容器以特权模式运行，或者添加了 SYS_TIME 能力，可以在容器内部修改时间，但是会影响宿主机的时间。
  2. 使用 libfaketime（推荐用于测试）
    libfaketime 是一个库，可以拦截系统调用，让应用程序看到虚假的时间，而不会真正修改系统时间。这种方法适合用于测试。 验证是否成功 若想要恢复，直接把环境变量修改为空即可 实际测试后虽然容器命令行内生效，但是服务无法读取到
    查询到原因是：faketime 只能对「动态链接」的 glibc 程序生效，而官方 OpenJDK 镜像（openjdk:8/11/17…）为了体积和兼容性，把 java 做成了静态链接的 musl 版本（Alpine）或静态 glibc 版本
    更改镜像打包的dockerfile 直接把环境变量的LD_PRELOAD和FAKETIME加到dockefile中会导致服务无法启动

Kimi的回答是:
JVM 对 LD_PRELOAD 非常敏感，尤其是涉及时间函数（如 clock_gettime、gettimeofday）时，faketime 会导致 JVM 崩溃或卡住，表现为：
· 容器启动后无日志
· java 进程不存在
· docker logs 无输出
